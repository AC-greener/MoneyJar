# Instructions 

## 思路
这是一个移动端记账软件，旨在帮助用户轻松管理个人财务。有三个页面：记账页面、统计页面和设置（我的）页面。
- 记账页面：用户可以在此页面记录每笔收入和支出，核心功能是语音输入，输入完成之后调用大模型转换成结构化数据来记账。语音输入完成之后会填入输入框，点击发送按钮即可完成记账。
- 统计页面：通过图表和数据分析，帮助用户了解自己的消费习惯和财务状况。
- 设置（我的）页面：用户可以在此页面管理个人信息、账户设置等。

## 技术栈
这是一个记账 H5 应用的技术栈：

---

**前端技术栈**

- **框架**：React 19 + Vite 6 + TypeScript，利用组件化思想提高代码复用性和可维护性
- **样式**：Tailwind CSS v4，采用响应式设计（移动端优先），确保应用在不同设备上良好展示
- **路由**：React Router v7，实现 SPA 页面导航与路由守卫（登录鉴权）
- **网络请求**：Axios，统一封装请求拦截器、响应拦截器及错误处理，与后端 API 进行数据交互
- **状态管理**：Zustand，管理用户信息、账单数据等全局状态
- **表单处理**：React Hook Form + Zod，处理收支录入、用户登录注册等表单校验
- **图表可视化**：Recharts，展示收支趋势、分类占比等统计图表
- **日期处理**：Day.js，处理账单日期格式化与区间筛选
- **构建 & 部署**：Vite 构建打包，产物部署于 H5 容器或移动端 
- 后端代码已经开发完成了，前端所有的接口都需要参考后端已经实现的API，后端代码在 `/Users/tongtong/AndroidStudioProjects/MoneyJar/server` 目录下。

**开发规范**

- ESLint + Prettier 统一代码风格
- 路径别名 `@/` 映射 `src/` 目录，提升模块引用可读性
- 环境变量通过 `.env` 文件管理，区分开发/生产环境接口地址

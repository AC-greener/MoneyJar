# 📝 【钱罐子 / MoneyJar】MVP 产品需求文档 (修订版)

## 1. 产品概述

**钱罐子** 是一款主打“无感记账”的智能财务管理工具。通过 AI 驱动的自然语言处理技术，将枯燥的手动录入转变为自然的对话交互，旨在为用户提供极速、智能且可靠的记账体验。

## 2. 典型用户场景 (User Scenarios)

### **场景 A：智能识别与自动分类 (Smart Inference)**

- **描述**：用户在超市买完菜，双手提着购物袋。
- **动作**：用户单手打开 App，长按录音按钮说：“买菜花了 35 块”。
- **结果**：App 实时将语音转为文字，后台 AI 自动识别出金额“35”和意图“买菜”，并自动归类为“生鲜”分类，用户无需任何手动选择即可完成记账。

### **场景 B：离线韧性与后台同步 (Offline Reliability)**

- **描述**：用户在地下停车场或信号不佳的区域打车。
- **动作**：用户打开 App 快速输入文字“打车 20 元”并点击保存。
- **结果**：App 识别到当前无网络连接，立即将账单加密存入本地安全库。当用户回到有信号区域时，App 在后台静默将数据同步至云端，确保数据永不丢失且多端一致。

### **场景 C：周视图与财务汇总 (Data Insight)**

- **描述**：用户在周末复盘本周开支。
- **动作**：用户进入“统计”页面查看“本周概览”。
- **结果**：App 提供一个清晰的时间轴列表，汇总展示本周每一笔支出，并在顶部显著位置标出“本周总支出”。通过简单的列表，用户能迅速掌握资金流向。

------

## 3. MVP 功能范围 (Scope)

### 3.1 核心录入系统

- **语音/文字双模输入**：支持自然语言描述。AI 需识别金额、备注，并根据预设逻辑判断收支属性。
- **智能自动分类**：系统需内置一套基础分类库（如：餐饮、交通、生鲜、数码、其他），AI 自动匹配最相关的标签。
- **即时反馈 UI**：识别完成后，UI 需以卡片形式展示解析结果，允许用户进行极秒的确认或手动微调。

### 3.2 数据持久化与同步

- **离线优先架构**：所有录入动作优先触发本地存储，保证在任何网络环境下“保存”操作无延迟。
- **增量同步机制**：网络恢复后自动将本地变更推送到服务端。

### 3.3 统计与展示

- **简易账单列表**：按日期倒序排列，支持无限滚动查看历史记录。
- **周期性汇总**：提供“本周”和“本月”的总支出统计数值。

------

## 4. 交互原则 (UX Principles)

- **极简操作**：核心功能（记账）应触手可及，避免多级菜单。
- **容错设计**：如果 AI 自动分类错误，用户应能通过一次点击快速修正。
- **视觉节奏**：列表展示应具有良好的呼吸感，大额支出应有视觉强调。

## 5. 性能与品质要求

- **解析耗时**：从输入完成到 AI 返回分类结果，感官延迟应低于 2 秒。
- **同步感知**：同步过程不应阻塞用户界面，应在后台静默完成。
